# 2026-05-03 — Bugfix: duplicate entries + always-buy

## Issues Fixed

### Bug 1: Interpreter always returns BUY side
**File:** `engine/interpreter.py` — `_check_entry()` method
**Root cause:** Side determination loop only checked if ANY entry condition was `CROSSUNDER`. If all conditions were `CROSSEOVER` (as in strategies 11 & 12), side stayed `"buy"` forever.
**Fix:** Now finds the primary crossover/crossunder condition and derives side from its type. CROSSEOVER → buy, CROSSUNDER → sell, no crossover → default buy.
**Commit:** `a654a16`

### Bug 2: Duplicate trade entries on every polling cycle
**File:** `engine/live_trading/live_engine.py` — `_evaluate()` method
**Root cause:** `existing_keys = set(self.order_manager._open_trades.keys())` returned trade_id strings like `"12345678"`, but compared against `f"{side}_{symbol}"` like `"buy_BTCUSD"`. These can never match → `if trade_key not in existing_keys` always True → opens duplicate trade every 60s.
**Fix:** Now uses `existing_open_sides = {t.side for t in self.order_manager._open_trades.values()}` → correctly prevents same-side duplicates. Also discards side from set when a trade closes, allowing fresh signals.
**Commit:** `a654a16`
